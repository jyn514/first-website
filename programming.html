Choosing a language

What is a programming language?

Basics

Computers, as we all know, understand only `on` and `off`, often stylized as 'zeros and ones' [1]. How then can we communicate with them, or do anything at all? A pre-built program, called a 'compiler' [2] and built by much smarter minds than I, turns logical statements - `Do this; if this condition is true, do that` - into binary code, instructions the computer can understand.

Compilers

What does this have to do with a language? The language you program in determines what statements you can make - you can only write code that a compiler can recognize and turn into binary statements. If you make a mistake, the compiler will throw an error. This can be anything from scrolling red text to a simple one-line message to a detailed breakdown of where your program went wrong.

Static and Dynamic

Although all programs eventually are compiled to binary code, the experience can be very different depending on the language. Dynamic languages[3] are more lenient - see here for an example. Beginners tend to gravitate towards them because they're quicker to pick up and allow more errors. Static languages are much less forgiving, but also do much more of the work for you.

Choosing

So, which language should you go with? Well, there are about 200 in existence, so that's not as easy to answer as it might seem. I recommend this site when first choosing. Don't worry, programming concepts tend to transfer between languages.

Basic Software

What software do you need to get started?

  1. A text editor

    This is perhaps the most important item here. Formatted text cannot be parsed by almost any compiler or browser. For general use, Notepad++ (for Windows) or gedit are excellent tools.

  2. Source control

    Source control is a way of keeping track of the changes you make to a document. See the git intro for more info. Git is one of the most popular versions of source control. You may also wish to upload to the cloud - GitHub integrates very well with git for this.

  3. A compiler

    This will vary based on your language. Here are some of the most common:

    LanguageCompilerNotes
    JavaOpenJDK As Java is one of the most popular languages, there are many different compilers. This one is an open-source and widely used.
    PythonOfficial website Python has very different syntax from most languages. Because of this, it's much easier to learn than most. Python 3 is the 'modern' version, while Python 2 is used in most legacy code. The two, unfortunately, are rarely compatible.
    C#Mono C# is a proprietary language owned by Microsoft. Mono is an open-source implementation.
    HTMLAny web browser HTML is not a programming language as such. It is the source code for every webpage on the modern web. Complete pages usually have CSS and JavaScript as well.
    C++GCC C++ is the modern version of C, one of the oldest languages still used and certainly the most influential. As with Java, many different implementations exist.
